# FAQ とトラブルシューティング

Q: TypeScript の型エラーでビルドが通らない

A: 一時的には該当箇所に `// @ts-ignore` や `any` を入れて回避できますが、正しい対応は型定義を修正することです。エラーの内容をよく読んで、該当する型（`types/`）を更新してください。

Q: 画像が表示されない

A: `public/` に置いたファイルは `/` で参照できます。Next の `next/image` を使う場合は `next.config.js` の `images.domains` 設定を確認してください。

Q: 新規記事がビルド時に生成されない

A: `src/app/.../generateStaticParams.ts` が記事一覧を返す仕組みになっているか確認してください。`content/` を読み込むロジックが正しく slug を返していることが必要です。

Q: 外部 API に接続できない／テスト用のモックを使いたい

A: 環境変数（例: `NOTION_ALLOW_MOCK`）でモックを切り替える実装があります。開発時は上の `how-to.md` にある起動コマンドを使ってください。

注意点:
- 重大なリファクタやルート名の変更を行うと型やリンク参照が壊れる可能性があります。小さな変更から始め、Git コミット単位で確認しましょう。
